DNS Poisoning

Modifying DNS Server

  • Well only possible if attacker has gain access to DNS server computer.

  • Attacker will modify and probably redirect to his own set up computer.

Modifying Host File

  • First of all if browser doesnot have cache, it will look up into /etc/hosts file to resolve DNS name and if attacker modifies info in that file, whenever user requested particular web page they will be presented as attacker webpage.html

Domain Hijacking

  • Gaining Access to Domain Resgistrar and modifying records to point to attacker IP.

  • This can be sophisticated because, records in DNS servers will auto be modified because once you update in registrarer, server will trust that registrar and will update record.

URL Hijacking / Typosquatting / Brandjacking

  • Attacker will have similar domain name as to of particular target organization.

  • Now what he could do is to redirect the traffic to different website where he will earn Ads revenue from.

  • Attacker could also redirect traffic to competitor of the target.

  • Attacker could even sell the domain name to the competitor.

  • Attacker may perform auto download or drive-by download.

  • Attacker my present a phishing site.
