Memory Injections

  • All the software to be able to run should load from disk to memory.

  • Once they are in memory they will have their own process to run or inject itself on another process.

  • If injected in another process, it will past from process based firewall or detection.

  • The software/malware will have same privilege as of process.

Scope of Memory based injections

  • Process
  • DLLs
  • Buffers
  • Threads
  • Memory Mgt Functions

DLLs

  • Windows library containing code and data.

  • Type of software or application that many different processes/apps can use.

  • Attacker install DLL

  • Attacker find process

  • Attacker will want DLLs to run as a part of that process

  • Attacker wil place path to DLL on that process

  • As process is running, when it moves to the point where it needs to access or derefernce that DLL address, it will go to that location/address and load into the process

  • Now Attacker could run many other apps as part of that DLL
