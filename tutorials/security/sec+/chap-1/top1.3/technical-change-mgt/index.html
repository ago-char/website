• Change control mechanism in POV of technician.

How to make change ?

  • Create Allow/Deny List.

  • Allow List must pass all the security checks and be Very Restrictive in nature.

  • Deny List contails known Vulns, malware, trojan carrying apps, etc.

  • Get well documented Change Control Docs from the Approval Department (be very restrictive), you could be flexible on the specific change you are asked to ?

    • Will you make change not related to given task if you have enough time and the change is necessary ?
    • Will you not make change if asked change required some pre-configured file be modified ?

  • Prevent Downtime when possible

    • Update when system is not running (maybe at holiday)
    • You may ask users move into secondary system and update on the primary system
    • You may automate the process of change if possible
    • Always have backout plan
    • Send Emails/Message

  • If it is leagacy system, understand the system before making any changes.

  • If your program requires Dependencies to be updated, update that first.

  • Change is almost Permanent, so must documentate so you could make further change in future.

    • It is always good to know what was previously changed before making any other change, it helps us and the user.

  • Track changes to file or config data for Version Control, you may use git if you are workinig with some code or program
