Encrypting Data

While is seems simple task to deploy encryption on the target system or organization, it is often necessary to look over certain points before doing so :

  • Know the necessity, use encryption only if needed and in security concern files

    • There is no point to use encryption on public available data
    • Encrypt files to be travelled over non secure tunnel

  • In organization, must of the data are structured and maybe stored on SQL database, you could apply encryption to the whole table or column based.

    • Whole table if all the columns contain security concerned data.
    • You could apply only to Licence_No. in the table of 'Name', 'Address', 'Licence_No'.

  • You may want to sacrifice speed for perforemance and more security and vice-versa for all, choose again according to necessity.

  • You may want to perform multiple level of hashing or encryption when security is prime concern for the application.

  • You could encrypt full disk also depending on the OS you use

    • Bitlocker - popular for "nt" systems
    • LUKS, dm-crypt - among "linux" systems
    • FileVault - for "mac"
    • VeraCrypt - maybe for "all"

  • Compromise speed, performence and security while choosing algorithm

  • You would also want different data be viewed for different users:

    • Say 'SSN' column be visible only for privilege users
    • For that you will install key only on those 'privilege' system, that will auto decrypt the column, for normal users data in 'SSN' column makes no sence, they might even don't know that it is 'SSN' column coz. column_name itself is encrypted.

  • You would like to increase N-bit in encryption depending on current technology state.

    • Machine being powerful, can may be brute force up to 64 bit or around of hash.
    • You would want to choose at least 128 bit key encryption.
    • In future gen of AI and Robots, Qunatum Computers, you would like to increase to a lot larger bit of encryption.

  • You probably want to increase the level of encryption, and vary the length and algorithm type for different level to hide your pattern

  • You could essentially enhance security combining encryption with Steganography tools

    • Image Steganography tools : steghide, openstego, imagehide
    • Audio Steganography tools : mp3stego, audiostegano, stegoaudio, deepsound
    • Video Steganography tools : videostegano, stegovideo
