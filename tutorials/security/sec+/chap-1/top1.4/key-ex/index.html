Key Exchange

Yes, we can generate keys on our own, but how would you exchange them over the internet ? This is not quite hard and not quite easy also as internet is the most unsecure medium of exchange for the information. Unfortunately, we don't have the luxuiry to visit on site to exchange key coz in the world of internet, where every miliseconds count, we can't afford delay. So, How ?

  • You could send key over encrypted session ?

  • Yes, but this is only for second key after the session is established

  • So, for exchange key over unsecure medium, you may use such medium where you both end will know key but without sending key, strange ? SEE:

    • 'A' creates private-pub key pair and send pub key to 'B' since public key is intended for public.
    • 'B' does the same
    • 'A' now uses : pub(B) + pri(A) = New_key1
    • 'B' now uses : pub(A) + pri(B) = New_key2
    • As, own pair of pub-pri key are mathematically interrelated, even with the computation of these different value will generate the same key and hence New_key1 = New_key2
    • As both are the same, it can be use for encryption and decryption
