Hash

  • Irreversible or one way function.

  • Short string representation of large data .

  • Use strong Hash.

  • Don't use md5 for much sensitive, Hash Collision was occured on 1996. i.e. same hash for 2 different docs.

  • It provides integrity, could also be use as digital signature.

  • Hash can be salted, Hashes of same password is always same, so use salt so that even if the password are identical, hashes will be different.

  • Rainbow Table is pre-computed hash-value table, use of salt could make Rainbow Table not much useful for attacket

Digital signatures]

  • Proof of Integrity.

  • Proof of the source of the message.

  • Creator will sign the doc/data with his private key to create digital signature.

  • User will verify with the creator's public key if sign is valid.
