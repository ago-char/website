Common Steganogaraphy Techniques

  • Network Based (embed messages within TCP Packets)

  • Use of multimedia (images, audios, videos)

  • Physical steganography with special kind of printers

  • Hiding info in publicly available docs, journals, etc.

Tokenization

  • Replacing sensititve data with non sensitive placeholder :

    • for eg: SSN of 222-44-9988 can become 559-23-9849 and can be used as Token, often these are one-time use tokens

  • Use different algorigthm to create Card Number Token, token will do the transaction and use per session only, so even if it will be on clear text, Malicious Actor in the middle could not be able to make purchase with this session token.

  • These token and original data can't be reversed-engineered, aren't math..lly related

Data Masking

  • Hiding sensitive part of data

  • For eg: in your shop bill, your last 4 of card info be visible and other as *

  • Restrict who should be dealing with such info, for eg: in banks, certain employees will only have access to sensitive data, other will have a piece of data just enough for customer validation and other process
