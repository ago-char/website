Protecting The DATA

Geographic Restrictions

One of the way to protect data is to apply geographic restrictions on that data on the basis of where the data is located and also where the user is located.

Network Location

  • Identify based on IP Subnet i.e if you are inside the private network or accessing through public or remote location.

GeoLocation

  • GPS co ordinates can be used to identify user's location which are very accurate.

  • You can also use 802.11 Wireless (wifi) databases, to identify Wifi the user might using, which is less accurate but can predict area if you know the scope.

  • IP Address, are also means but not reliable, but if you have enough resources like how ISP distribute IP Address, you may be able to determine the scope of location.

GeoFencing

  • Once you have identified geolocation and network location or may be have certain idea where user might be, you can now apply certain rules and policies regarding geofancing.

  • For eg : if user is watching netflix from Europe, you may wanna restrict certain Asian content, or recommend Europe Cinemas, etc.

  • You may wanna grant user access to data if he is inside one of your corporate building on any branch, but not from home.

Segmentation

  • If you work in a type of organization, where everything is stored on single database, you might wanna request to change the structure.

  • For eg : if info regarding username, passwords, personal information, credit cards of user are all at single location, and if attacker was able to gain access to that database, he can transfer and make public the data in one go. But if it was to be segmented, it would be hard for him to collect data.

  • Segmentation, also helps in applying certain rules or security mechanisms for certain type of data. For eg : usernames be in plain, but SSN be encrypted or masked.

Encryption

  • Encrypt data in use, transit or rest.

Hashing

  • Associate hash with the type of data you're sending.

  • Hashing is mostly used in downloading binaries or source from internet. The vender will post it's hash, after downloading, you will want to calculate hash, if matches you are good to go, else, you may be wiretapped and downloading may have been breached.

Obfuscation

  • Hiding piece of info into another un useful info like secret data in image, etc.

  • Developer may write code to interpret simple string like "hey bob", differently.

Masking

  • A type of obfuscation where sensitive part of data is made invisible, or encrypted or replaced by something. For eg : AAA~~~~~~~~~ i.e AAA might be replaced or as it is, others are hidden from the lottery ticket.

Tokenization

  • Replace sensitive data with non-sensitive placeholder.

  • Create token for certain credit card transaction.

  • If attacker intrecept the connection, and try to make pursace, he won't be able to do so as it is only ONE TIME USE.
