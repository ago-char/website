Software Defined Networking (SDN)

  • When we talk about Networking, we may think of switches, routers, etc. and ports, cable, etc. inside them and that is true.

  • But these device are doing a lot more in themself.

  • They can be catagorized into different planes of operation.

Layers/Planes

Infrastructure Layer / Data Plane

  • Processes the netwrok frames and packets

  • The operation like : forwarding, NAT, tagging , trunking, encryption

Control Layer / Control Plane

  • Define how the data plane with transfer data from point A to point B.

  • This can be accomplished with the help of : VLAN tables, routing tables, session, DHCP config, NAT tables, etc.

Application Layer / Management Plane

  • Defines how to actually control and manage the device

  • SSH, browser, API could be the medium of doing so

Why SDN ?

  • As all the functionality can be done within software, ultimately they all are logics.

  • Logics can be defined on configuration files.

  • Control can be defined based on database.

  • Management can be defined on the console apps.

  • As all is done on software, it makes CLOUD, or VM technology possible.

  • Now security only depends on file, physical security is ommited.

  • As all files are combined, we can dynamically create a network on the cloud.

  • Backup files, delete the topology.

  • Restore file, topology is as it as.
