Firewalls and Types

  • Firewalls are used to control the flow of traffic between endpoints.

  • Firewalls have different access control mechanisms.

  • Since firewall is watching all traffic watching through it, it is a perfect place to provide additional security controls like Antivirus or Anti-Malware.

Network-based Firewalls

Read basics : Network Firewall

  • Traditionally, these firewalls were (still are) configured to work on OSI Layer4 - Transport Layer i.e. rules were configured on TCP port or UDP port.
  • Modern Firewalls (Next Generations Firewalls) are more advanced that can work at Application Layer which provides more control and easy to configure on protocol basis like : SSH traffic, HTTP traffic, etc.
  • These firewalls may have additional capacity of encrypting traffic (VPN), routing (NAT) and so on.

Next Generation Firewall

  • NGFWs works at Application Layer of OSI Model.

  • They are capable of advanced decoding and analyzing.

  • Since they work on Application and can dedode various IP frames and packets, they are able to control traffic flow on the basis of specific protocol and Application.

  • For eg: if old firewalls or port based firewall were able to block connection at 443 port, NGFWs are capable of blocking youtube but allowing twitter and likely so.

  • Expanding, the capability to filter URLs, also work as Intrusion Prevention System.

Web Application Firewall

  • Web Application Firewall is a specific form of application firewall that filters, monitors, and blocks HTTP traffic to and from a web service.

  • They applies rules to HTTP/s conversations.

  • Often rules are written on the basis of input, because it is the method which most attackers target to exploit website or web app.

  • This type of firewall may not deployed by all type of organizations, but for some organizations like : Payment Card Industry Data Security Standard (PCI DSS), this seems mandetory.
