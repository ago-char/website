  • You might configure the zones as :

    • Trusted, Untrusted
    • Internal, External
    • Inside, Internet, Servers, Databases, Screened

Attack Surface

  • Refers to the entry point for attacker, that can be :

    • Application Code
    • Open Ports
    • Authentication Process
    • Human Error

  • You may want to :

    • Review code
    • Close unwanted ports, port based firewall rules
    • Patch Auth Process
    • Monitor and log your traffic to reduce attack surface.

Connectivity

  • It is a part of Attack Surface.

  • Our internal devices are (may be) somehow connected to eachother.

  • So, you should consider :

    • Physically securing cables (don't leave open cable ports, nice wiring)
    • Application Level Encryption (SSH, HTTPS)
    • Network Level Encryption (VPN, IPsec tunnels)
