Useful Network Appliances

Jump Server

  • If require to talk to outside network's server, you may not directly have access to that server, you will talk to jump server which sits between outside server and your netwrok, once you connect to jump server, then you can rdp or ssh to the actual server only from that jump server.

  • Jump Servers are hardened and limited access and can be accessed with SSH, Tunnel or VPNs or RDP, etc.

Porxy Server

  • Sits between you and internet or another server where all requests are forwarded to proxy server.

  • Proxy server than forwards the request to intended server, analyze it if it is malicious or not, may perform filtering like (url filtering), caching, also some access control and etc. and finally forward the reply to you.

  • There is regular proxy whcih should be explicitly mentioned in end users, but also transparent proxy of which user has no idea about, but data are being transferred back and forth between that proxy and client/user.

Application Proxy

  • could be NAT proxy used on routers to convert ip addresses.

  • application level proxy only understands a single or a single set of protocols, for ex : a proxy could only understand ftp and other may understand http, https and ftp too.

Forward Proxy

  • Sits between you and internet for outgoing traffic.

  • Your request will be forwarded to that proxy, which will sent to the internet and receives the reply, if it is legitamate enough to forward you after analyzing, you will get the reply.

Reverse Proxy

  • Sits between you and internet for incomming trafffic.

  • Suppose, you have web server, instead of making it internet facing, you could place reverse proxy where users from the internet will request to your web site.

  • Those requests are now evaluated, droped, rejected or forwarded to your server and then the server will provide the response.

  • This provides additional security to your web server because those malicious requests aer handled by proxy instead of your web server.

  • That proxy server can also act as caching, so that it will reduce communication for same requests between you and your proxy.

Open Proxy

  • Available freely on the internet for any one to use.

  • Commonly used to circumvent (avoid, get around, bypass) the existing security controls.

  • You might find your internal proxy too slow or has a lot of ACLs and now you configure, to use external one for better result.

  • But these are managed by some third parties, which we may not have any idea about. They might be configured on such a way that they will do the same task but also send some malicious traffic to you or may collect your information for sale or advertisements or else.

  • You should not use those proxies, unless you know what exactly you are trying to achieve from that.

  • On many of the companies, these known bad/good open proxies are blocked in order to enhance security.

Load Balancer

  • Device connected to servers (identical servers).

  • This will manage load across servers.

  • Instead of sending all the traffic to one server, it will send to multiple servers.

  • TCP offload : Instead of creating multiple tcp connection to server multiple clients, it will connect to one tcp connection for each server and offload or track packets for each clients in single connection.

  • SSL offload : Instead of servers doing all the encryption or decryption, for incomming traffic load balancer will perform that decryption and will communicate to servers in the clear text, for outgoing traffic server will again send in clear but now will be encrypted by the load balancer. It helps servers' load to reduce.

  • There is often purpose build hardward inside the load balancer to provide efficient encryption and decryption processes.

  • Load balancers can also acts as proxies to filter, cache, content control, content switching, and so on just like proxy. It may also perform QoS, Prioritization, etc.

Active/Active Load Balancer

  • All the server connected to load balancers are active and the load will be distributed to all the active devices by the load balancer.

Active/Passive Load Balancer

  • Some of the servers connected to load balancers will be active and some will be in passive or standby state.

  • Load will again be distributed to active ones.

  • If active one fails, or are out of service; load balancer will automatically reconize and now the passive ones at standby position will be ready to server. That failure server will be maked as "no longer available" until they are fixed or become availabe, that will too be recongnized by the balancer.

Sensors and Collectors

  • Many of the devices, you already using like switches, routers, firewalls may have built in sensors and collectors to them.

  • If these devices found any unusual or even usual traffic that will be logged for later analysis.

  • IDS and IPS will generates different data, firewall logs, auth logs, web server access logs, database, email logs etc. all are useful data sensed by these devices.

  • More often, in companies, they may place such device which sole purpose is to sense the activity and log or collect them.

  • All these logs are usually sent to 'proprietary consoles', SIEM consoles, syslog servers, etc. centrally for central analysis and detection.

  • These event captures may not only be sent periodically, but also in real time in security concerned areas for instant threat control.
