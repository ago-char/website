Secure Baselines

  • Not only the application you are deploying be secure, but also the underlying baselines i.e. OS, vendor devices, firewall, etc. should be deployed or maintained to the best security practice.

  • These should be performed often and create report regarding the check.

Establish Baselines

  • For deploying secure baselines, you first need to build one.

  • Fortunately, these baselines are often vendor specific. For eg: if you are using windows OS, they have well documented security baselines practice, and often similar with any product if it is switches, routers, software, etc.

  • For that, you sould consider visiting specific manufacturer for reference.

  • There are many policy regarding everthing, but you should filter security-related.

Deploy Baselines

  • Once you build baselines for security, you gonna have to deploy them.

  • Often managed by centrally administrative console or utility like iptables for linux based system as firewalls, or so on.

  • This may not be efficient and effective when in large no. of hosts, you may need multiple deployment mechanisms like : Active Directory policy, group policy, MDM, etc.

  • While managing these is kind of tedious and overwhelming task, you should always look forward to automation when possible.

Maintain baselines

  • Security baselines are often best practices, but should regularly maintain them.

  • Security baselines may require update if :

    • A new vulnerability is discovered
    • Application itself has updated
    • New OS is installed/deployed

  • This may not always be easy, because sometimes, existing things like software or hardware or even OS version, may not interact well with the application. You may require it to further modify, update or even change underlying infrastructure.

  • Audit and Documentation is most.
